# TransGPT 文案优化技能包

这是一个 Claude Code Skill，用于优化 TransGPT 产品的各类文案。

## 功能简介

这个技能包可以帮助你：
- ✅ 优化 App 内的按钮、提示、说明文字
- ✅ 改写营销文案和宣传语
- ✅ 生成多语言版本的文案（中英双语）
- ✅ 提供多个优化方案供选择
- ✅ 符合 TransGPT 品牌调性

## 如何使用

### 方法 1：通过 slash 命令（推荐）

```
/copywriting 优化这段文案：...
```

### 方法 2：直接对话

Claude 会自动识别文案优化需求，直接说：

```
帮我优化这段文案：
"您的免费额度即将用完..."
```

或

```
这个按钮的文字不太好，帮我改一下：
"立即购买会员"
```

### 方法 3：提供更多上下文

为了得到更好的优化建议，你可以提供：

```
帮我优化首页的欢迎语。
现在的文案是："Hi, I'm TransGPT. I make Live translation easy."
这段文字会显示在首页顶部，用户第一次打开 App 就能看到。
希望文案更友好、更易懂。
```

## 输出格式

每次优化会给出 2-3 个方案，格式如下：

```
### Option A (Recommended)

**中文版**：
"优化后的文案" (X 字)

**English**：
"Optimized copy" (X words)

✅ 优点说明
✅ 优点说明
⚠️ 注意事项

### Option B

**中文版**：
"另一个版本" (X 字)

**English**：
"Another version" (X words)

✅ 优点说明
⚠️ 不足之处

### 推荐理由

推荐 Option A，因为：
1. 理由一
2. 理由二
```

## 优化原则速查

1. **简短易懂**
   - 中文 ≤ 15 字
   - 英文 ≤ 10 词
   - 避免专业术语

2. **无歧义**
   - 明确具体
   - 避免模糊表达

3. **用户视角**
   - 使用"你"
   - 关注用户利益

4. **行动导向**
   - 动词开头
   - 明确指引

5. **多语言**
   - 必须提供中英双语
   - 标注字数/词数

## 常见使用场景

### 场景 1：按钮文案
```
这个升级按钮的文案需要优化：
"购买 Pro 会员"
```

### 场景 2：弹窗提示
```
用户额度用完时的提示：
"您的免费使用次数已用完，请升级会员继续使用"
```

### 场景 3：功能说明
```
首页的功能描述太长了：
"TransGPT 提供实时语音翻译、视频翻译、拍照翻译等多种翻译方式"
```

### 场景 4：错误提示
```
网络错误时显示：
"Network error. Please try again."
```

### 场景 5：营销文案
```
App Store 的产品描述第一段需要吸引人
```

## 最佳实践

### ✅ 做这些

1. **提供上下文**
   - 说明文案出现的位置
   - 说明想要达到的效果

2. **明确需求**
   - 有字数限制吗？
   - 有语气要求吗？
   - 有特殊场景吗？

3. **对比选择**
   - 查看所有方案
   - 选择最适合的
   - 可以要求调整

### ❌ 避免这些

1. **不要只给一个词**
   - ❌ "优化'购买'"
   - ✅ "优化这个按钮文案：'购买会员'"

2. **不要缺少上下文**
   - ❌ "这段话怎么说更好"
   - ✅ "首页顶部的欢迎语怎么说更好"

3. **不要要求与品牌不符的风格**
   - ❌ "写得夸张一点，震撼一点"
   - ✅ "写得友好一点，自然一点"
